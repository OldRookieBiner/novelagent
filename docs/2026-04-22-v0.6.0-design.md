# v0.6.0 设计文档

## 概述

v0.6.0 版本包含两大功能模块：

| 模块 | 内容 |
|------|------|
| **模块 A：模型配置增强** | 自定义添加模型、健康检查、启用/停用管理 |
| **模块 B：大纲生成流程优化** | 直接生成、停止生成、返回确认、修改灵感采集 |

## 模块 A：模型配置增强

### 1. 数据库设计

新增 `model_configs` 表：

```sql
CREATE TABLE model_configs (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id) ON DELETE CASCADE,
    name VARCHAR(100) NOT NULL,           -- 显示名称，如 "我的 DeepSeek"
    provider VARCHAR(50) NOT NULL,        -- 预设标识或 "custom"
    base_url VARCHAR(500) NOT NULL,       -- API 基础地址
    model_name VARCHAR(100) NOT NULL,     -- 模型名称
    api_key_encrypted TEXT,               -- 加密的 API Key（每个模型可独立配置）
    is_enabled BOOLEAN DEFAULT true,      -- 是否启用
    is_default BOOLEAN DEFAULT false,     -- 是否为默认模型（单选模式）
    last_health_check TIMESTAMP,          -- 上次健康检查时间
    health_status VARCHAR(20),            -- healthy / unhealthy / unknown
    health_latency INTEGER,               -- 延迟毫秒数
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 确保每个用户只有一个默认模型
CREATE UNIQUE INDEX idx_model_configs_user_default 
ON model_configs (user_id) 
WHERE is_default = true;
```

**预置数据**：用户首次访问模型配置页面时，自动创建 3 个预设模型（不填 API Key）：
- DeepSeek (火山方舟)
- DeepSeek (官方)
- OpenAI

### 3. 健康检查实现

发送一个简单的 chat 请求测试连通性：

```python
async def check_model_health(base_url: str, api_key: str, model_name: str) -> dict:
    """检查模型健康状态"""
    start_time = time.time()
    try:
        client = AsyncOpenAI(api_key=api_key, base_url=base_url)
        # 发送最小请求
        response = await client.chat.completions.create(
            model=model_name,
            messages=[{"role": "user", "content": "Hi"}],
            max_tokens=5
        )
        latency = int((time.time() - start_time) * 1000)
        return {"status": "healthy", "latency": latency}
    except Exception as e:
        return {"status": "unhealthy", "error": str(e), "latency": None}
```

## 模块 B：大纲生成流程优化

### 1. 灵感采集后直接生成大纲

**当前流程**：
```
灵感采集 → 点击"确认" → 跳转到大纲页面 → 再点"生成大纲"按钮
```

**优化后流程**：
```
灵感采集 → 点击"生成大纲" → 直接开始生成
```

**实现要点**：
- 修改 `InspirationForm` 组件，按钮文案从"生成灵感模板"改为"生成大纲"
- 点击后直接调用 `POST /api/projects/:id/outline` 开始生成
- 生成过程中显示停止按钮

### 2. 停止生成按钮

**实现方案**：
- 前端维护 `AbortController` 实例
- 点击"停止生成"时调用 `controller.abort()`
- 后端 SSE 流中断后，已生成的内容保留

**前端代码示例**：

```typescript
const [isGenerating, setIsGenerating] = useState(false)
const abortControllerRef = useRef<AbortController | null>(null)

const handleStartGenerate = async () => {
    abortControllerRef.current = new AbortController()
    setIsGenerating(true)
    
    try {
        const response = await fetch(`/api/projects/${projectId}/outline`, {
            method: 'POST',
            signal: abortControllerRef.current.signal
        })
        // 处理 SSE 流...
    } catch (err) {
        if (err.name === 'AbortError') {
            // 用户主动停止，保留已生成内容
            console.log('生成已停止')
        }
    } finally {
        setIsGenerating(false)
    }
}

const handleStopGenerate = () => {
    abortControllerRef.current?.abort()
}
```

**后端处理**：
- SSE 流中断时，检查已生成的 `accumulated_content`
- 如果有内容，解析并保存到大纲
- 更新项目阶段为 `outline_confirming`

### 3. 返回列表确认弹窗

**弹窗设计**：
```
标题：返回列表
内容：当前正在生成大纲，返回列表会中断生成进度，已生成的内容将会保留。
按钮：[继续生成] [确认返回]
```

**实现要点**：
- 在 `ProjectDetail.tsx` 的返回按钮添加条件判断
- 如果 `isGenerating` 为 true，显示确认弹窗
- 确认后停止生成并返回列表

### 4. 修改灵感采集记录

**触发条件**：`outline.confirmed === false`

**实现要点**：
- 在 `StepNavigation` 组件中，当大纲未确认时，"灵感采集"步骤可点击
- 点击后跳转到灵感采集表单，预填充已保存的数据
- 修改后保存，需重新生成大纲

**状态流转**：
```
大纲未确认 → 点击"灵感采集" → 修改表单 → 保存 → 阶段回到 outline_generating
```
