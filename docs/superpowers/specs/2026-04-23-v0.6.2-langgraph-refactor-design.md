# v0.6.2 LangGraph 全栈重构设计

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**版本：** v0.6.2

**目标：** 使用 LangGraph 重构小说创作工作流，实现状态累积、暂停/恢复、统一的工作流管理。

**技术栈：** LangGraph 0.2+ / FastAPI / PostgreSQL / React / Zustand / SSE

---

## 一、设计决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 重构范围 | 全栈（Agent + API + 前端） | 统一架构，最佳用户体验 |
| 工作流模式 | B+C 混合（设置可选） | 效率与控制平衡 |
| 暂停/恢复 | 项目列表状态 + 进入时提示 | 用户可控，不会忘记进度 |
| 状态存储 | PostgreSQL 数据库 | 数据安全，多设备同步 |
| 流式输出 | SSE（保持现有） | 改动小，与 LangGraph 天然配合 |
| 前端状态 | 完全重构 Zustand Store | 商业化友好，与后端对齐 |

## 二、LangGraph 图结构

```
[收集灵感信息] → [生成大纲] → [等待确认] → [生成章节大纲] → [等待确认]
                                                              ↓
                                                    [生成章节正文] ←──┐
                                                              ↓      │
                                                        [审核章节]    │
                                                              ↓      │
                                              ┌───────── 条件边 ─────┤
                                              ↓                    │
                                         通过: 下一章              │
                                              ↓                    │
                                        [有下一章？] ──────────────┘
                                              ↓ 完成
                                        [完成]
                                              
                                         不通过: 重写/用户决定
                                              ↓
                                        [重写章节正文]
```

### 节点定义

| 节点名称 | 功能 | 输入 | 输出 |
|----------|------|------|------|
| `collect_inspiration` | 收集灵感信息 | 用户表单 | collected_info, inspiration_template |
| `generate_outline` | 生成大纲 | collected_info | outline_* 字段 |
| `wait_outline_confirm` | 等待大纲确认 | outline | confirmation |
| `generate_chapter_outlines` | 生成章节大纲 | outline | chapter_outlines |
| `wait_chapter_outlines_confirm` | 等待章节大纲确认 | chapter_outlines | confirmation |
| `generate_chapter` | 生成章节正文 | chapter_outline, written_chapters | chapter_content |
| `review_chapter` | 审核章节 | chapter_content | review_result |
| `wait_review_decision` | 等待审核决定（条件边） | review_result | decision |
| `rewrite_chapter` | 重写章节 | review_result | chapter_content |

## 三、State 结构

```python
# backend/app/agents/state.py

from typing import TypedDict, Optional, Annotated
from operator import add

class NovelState(TypedDict):
    """小说创作状态 - 与前端 store 对齐"""
    
    # ========== 基本信息 ==========
    project_id: int
    
    # ========== 阶段控制 ==========
    stage: str  # inspiration | outline | chapter_outlines | writing | review | complete
    
    # ========== 灵感/输入 ==========
    collected_info: dict           # 用户填写的灵感信息
    inspiration_template: str      # 灵感模板
    
    # ========== 大纲 ==========
    outline_title: str
    outline_summary: str
    outline_plot_points: list[dict]  # [{order, event, conflict, hook}]
    outline_characters: list[dict]   # [{name, role, personality, motivation, arc}]
    outline_world_setting: dict      # {era, core_rules, power_system}
    outline_emotional_curve: str
    outline_confirmed: bool
    
    # ========== 章节大纲 ==========
    chapter_count: int
    chapter_outlines: list[dict]     # [{chapter_number, title, scene, ...}]
    chapter_outlines_confirmed: bool
    
    # ========== 章节正文（累积）==========
    written_chapters: Annotated[list[dict], add]  # [{chapter_number, content, word_count}]
    current_chapter: int             # 当前正在写的章节号
    
    # ========== 审核/重写 ==========
    review_mode: str                 # step_by_step | hybrid | auto
    review_result: Optional[dict]    # {passed, scores, issues, feedback}
    rewrite_count: int
    max_rewrite_count: int
    
    # ========== 工作流控制 ==========
    waiting_for_confirmation: bool   # 是否在等待用户确认
    confirmation_type: Optional[str] # outline | chapter_outlines | review_failed
```

## 四、数据库设计

### 4.1 新增表

```sql
-- 工作流检查点表
CREATE TABLE workflow_checkpoints (
    id SERIAL PRIMARY KEY,
    project_id INTEGER NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
    thread_id VARCHAR(100) NOT NULL,          -- LangGraph 线程 ID
    checkpoint JSONB NOT NULL,                 -- 完整的 State JSON
    created_at TIMESTAMP DEFAULT NOW(),
    
    UNIQUE(project_id, thread_id)
);

-- 索引
CREATE INDEX idx_checkpoints_project ON workflow_checkpoints(project_id);
CREATE INDEX idx_checkpoints_thread ON workflow_checkpoints(thread_id);
```

### 4.2 项目表新增字段

```sql
ALTER TABLE projects ADD COLUMN workflow_mode VARCHAR(20) DEFAULT 'hybrid';
-- workflow_mode: step_by_step | hybrid | auto
```

## 五、API 设计

### 5.1 新增 API

| 方法 | 路径 | 功能 | 说明 |
|------|------|------|------|
| POST | `/api/projects/{id}/workflow/run` | 启动/恢复工作流 | SSE 流式输出 |
| POST | `/api/projects/{id}/workflow/confirm` | 确认当前节点 | 继续执行 |
| POST | `/api/projects/{id}/workflow/cancel` | 取消当前操作 | 返回上一检查点 |
| GET | `/api/projects/{id}/workflow/state` | 获取当前状态 | 用于恢复页面 |

### 5.2 SSE 事件格式

```
event: node_start
data: {"node": "generate_outline", "stage": "outline"}

event: chunk
data: "大纲内容..."

event: node_done
data: {"node": "generate_outline", "result": {...}}

event: checkpoint
data: {"stage": "outline", "waiting_for_confirmation": true}

event: waiting
data: {"type": "outline_confirm"}

event: done
data: {"stage": "complete"}
```

```typescript
interface WorkflowState {
  // 阶段
  stage: 'inspiration' | 'outline' | 'chapter_outlines' | 'writing' | 'review' | 'complete'
  
  // 大纲
  outline: {
    title: string
    summary: string
    plotPoints: PlotPoint[]
    characters: Character[]
    worldSetting: WorldSetting
    emotionalCurve: string
    confirmed: boolean
  } | null
  
  // 章节大纲
  chapterOutlines: ChapterOutline[]
  chapterOutlinesConfirmed: boolean
  
  // 已写章节（累积）
  writtenChapters: WrittenChapter[]
  currentChapter: number
  
  // 审核
  reviewResult: ReviewResult | null
  rewriteCount: number
  
  // 工作流控制
  isRunning: boolean
  waitingForConfirmation: boolean
  confirmationType: 'outline' | 'chapter_outlines' | 'review_failed' | null
  
  // SSE 状态
  currentChunk: string
  currentNode: string | null
  
  // Actions
  setStage: (stage: string) => void
  updateOutline: (outline: Partial<Outline>) => void
  addWrittenChapter: (chapter: WrittenChapter) => void
  setWaitingForConfirmation: (type: string | null) => void
  appendChunk: (chunk: string) => void
  reset: () => void
  
  // 异步 Actions
  runWorkflow: (projectId: number) => Promise<void>
  confirm: (projectId: number) => Promise<void>
  fetchState: (projectId: number) => Promise<void>
}
```

## 七、工作流模式配置

### 7.1 设置页面新增

```
┌─────────────────────────────────────────────────────────┐
│ 创作流程模式                                             │
├─────────────────────────────────────────────────────────┤
│                                                         │
│ ○ 逐步确认模式 (step_by_step)                           │
│   每个步骤完成后暂停，等待您确认                         │
│   适合：想要完全控制每一步的用户                         │
│                                                         │
│ ● 智能混合模式 (hybrid) - 推荐                          │
│   大纲和章节大纲需要确认，正文自动生成                   │
│   适合：希望效率与质量平衡的用户                         │
│                                                         │
│ ○ 全自动模式 (auto)                                     │
│   一键完成，仅在审核不通过时暂停                         │
│   适合：信任 AI，追求效率的用户                          │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 7.2 模式行为对照

| 节点 | step_by_step | hybrid | auto |
|------|--------------|--------|------|
| 大纲生成后 | ⏸️ 暂停确认 | ⏸️ 暂停确认 | 自动继续 |
| 章节大纲生成后 | ⏸️ 暂停确认 | ⏸️ 暂停确认 | 自动继续 |
| 章节正文生成后 | ⏸️ 暂停确认 | 自动继续 | 自动继续 |
| 审核不通过时 | ⏸️ 暂停确认 | ⏸️ 暂停确认 | ⏸️ 暂停确认 |
