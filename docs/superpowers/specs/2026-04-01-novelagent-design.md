# NovelAgent 设计文档

## 项目概述

**NovelAgent** - AI 小说创作 Agent 系统，三 Agent 协作完成小说创作流程。

### 核心目标

- 初版专注跑通流程，验证 Agent 协作可行性
- 采用"先 Agent 后框架"策略，避免过度设计
- CLI 命令行交互，最简单直接的实现方式

---

## Agent 组成

### 三 Agent 流程

```
用户 ←──→ 大纲Agent（对话收集想法 → 大纲 → 卷纲 → 单元纲 → 章节纲）
                ↓ 确认后交付
         写作Agent（章节正文）
                ↓
         审核Agent（一致性 + 质量 + AI味 + 规则检查）
                ↓ 问题则重写，通过则继续下一章
```

### 大纲 Agent

职责：
1. 与用户对话，收集小说想法和细节
2. 自动询问补充信息（人物、背景、风格等）
3. 用户确认信息充足后，生成小说大纲
4. 与用户讨论修改大纲
5. 大纲确认后，生成所有卷纲
6. 逐卷生成单元纲，每卷确认后生成章节纲
7. 管理完整的创作状态

### 写作 Agent

职责：
1. 接收章节纲
2. 生成章节正文
3. 交付审核 Agent
4. 根据审核反馈重写

### 审核 Agent

职责：
检查章节正文的：
- **一致性**：人物名、地名、前后情节是否一致
- **质量**：文笔、节奏、逻辑
- **AI味检测**：是否有明显 AI 生成痕迹
- **规则检查**：是否符合用户设定的风格/限制条件

发现问题直接罗列，确认后让写作 Agent 重写，直到通过。

## 完整创作流程

```
1. 信息收集阶段
   用户对话 → 收集想法 → 补充细节 → 确认信息充足

2. 大纲生成阶段
   生成大纲 → 确认/修改 → 大纲定稿

3. 卷纲生成阶段
   一次性生成所有卷纲 → 确认/修改 → 卷纲定稿

4. 单元循环阶段（按卷循环）
   for each 卷:
       生成该卷单元纲 → 确认/修改 → 单元纲定稿
       
       for each 单元:
           生成该单元章节纲 → 确认/修改 → 章节纲定稿
           
           for each 章节:
               写作Agent生成正文 → 审核Agent检查
               → 有问题则重写，通过则继续下一章
           
           单元完成
       
       卷完成

5. 小说完成
```


## 阶段状态流转

```
collecting_info        # 信息收集
    ↓
outline_generating     # 生成大纲
    ↓
outline_confirming     # 确认大纲
    ↓
volumes_generating     # 生成卷纲
    ↓
volumes_confirming     # 确认卷纲
    ↓
unit_generating        # 生成单元纲（当前卷）
    ↓
unit_confirming        # 确认单元纲
    ↓
chapter_outline_generating  # 生成章节纲（当前单元）
    ↓
chapter_outline_confirming  # 确认章节纲
    ↓
chapter_writing        # 写章节正文
    ↓
chapter_reviewing      # 审核章节
    ↓ (通过)
chapter_completed      # 章节完成
    ↓ (还有下一章)
chapter_writing        # 继续写下一章
    ↓ (单元完成，还有下一单元)
unit_generating        # 下一单元
    ↓ (卷完成，还有下一卷)
unit_generating        # 下一卷的单元
    ↓ (全部完成)
completed              # 小说完成
```

## Prompt 模板设计

### 卷纲生成 Prompt

```
你是一个专业的小说大纲策划师。根据已有的小说大纲，将其拆分为若干卷。

要求：
1. 根据故事的起承转合自然分卷
2. 每卷有明确的主题和冲突
3. 卷之间有过渡和悬念

小说大纲：
{outline}

请生成所有卷纲，格式如下：
---
第X卷：[卷名]
概述：[本卷主要内容，100-200字]
核心冲突：[本卷的主要矛盾]
关键事件：[本卷的关键情节点，3-5个]
结局：[本卷如何收尾]
单元数：[数量]
---
```

### 单元纲生成 Prompt

```
你是一个小说大纲策划师。根据卷纲，将其拆分为若干单元。

要求：
1. 每单元是一个相对独立的故事单元
2. 单元之间有连贯性
3. 每单元包含若干章节

当前卷纲：
{volume_outline}

小说大纲参考：
{novel_outline}

请生成该卷的所有单元纲，格式如下：
---
第X单元：[单元名]
概述：[本单元主要内容]
章节数：[数量]
---
```

### 章节纲生成 Prompt

```
你是一个小说章节策划师。根据单元纲，生成每个章节的详细大纲。

要求：
1. 每章有明确的场景、人物、情节
2. 章节之间有连贯性
3. 每章有冲突和结局（或悬念）

当前单元纲：
{unit_outline}

卷纲参考：
{volume_outline}

请生成该单元的所有章节纲，格式如下：
---
第X章：[章节名]
场景：[发生地点]
人物：[出场人物]
情节：[本章主要情节，100-200字]
冲突：[本章的冲突/矛盾]
结局：[本章如何收尾/悬念]
预计字数：[字数]
---
```

### 章节正文生成 Prompt

```
你是一个小说作家。根据章节大纲，写出完整的章节正文。

要求：
1. 严格遵循章节纲的情节发展
2. 保持人物性格一致
3. 文笔流畅，有代入感
4. 避免AI生成的常见问题（过于书面化、过度解释、缺乏细节）

章节大纲：
{chapter_outline}

前文参考（上一章结尾）：
{previous_chapter_ending}

小说设定：
{novel_settings}

请写出完整的章节正文。
```

### 审核章节 Prompt

```
你是一个小说编辑。审核章节正文的质量。

审核维度：
1. 一致性：人物名、地名、前后情节是否一致
2. 质量：文笔、节奏、逻辑是否合理
3. AI味：是否有明显AI生成痕迹（过于书面化、重复表达、缺乏细节）
4. 规则：是否符合用户设定的风格

章节大纲：
{chapter_outline}

章节正文：
{chapter_content}

小说设定：
{novel_settings}

请审核并回复：
---
【审核结果】通过/不通过
【问题列表】（如果有）
1. [问题描述]
2. ...
【修改建议】（如果有）
[具体建议]
---
```

## 技术栈

| 层 | 技术 | 说明 |
|---|---|---|
| 语言 | Python 3 | AI 生态最成熟 |
| LLM 调用 | 直接 API 调用 | 无框架依赖，完全可控 |
| CLI | argparse + input() | 最简单交互 |
| 模型 | DeepSeek（初版） | 可切换，设计成多模型支持 |
| 持久化 | JSON 文件 | 人类可读，备份方便 |

## 验收标准

### 阶段1（已完成）
1. ✅ CLI 可启动，进入对话模式
2. ✅ Agent 能与用户多轮对话收集信息
3. ✅ Agent 能自动判断信息充足度并询问补充
4. ✅ Agent 能生成小说大纲
5. ✅ 用户可提出修改意见，Agent 能调整大纲
6. ✅ 状态持久化到 JSON 文件，可断点续聊
7. ✅ 支持查看项目列表和状态

### 阶段2（完整流程）
1. 大纲确认后能生成所有卷纲
2. 用户可修改卷纲
3. 每卷能生成单元纲
4. 每单元能生成章节纲
5. 写作Agent能生成章节正文
6. 审核Agent能检查章节质量
7. 能根据审核反馈重写
8. 完整流程能跑通：大纲→卷纲→单元纲→章节纲→正文→审核→完成

## 风险与对策

| 风险 | 对策 |
|------|------|
| LLM 回复不稳定 | Prompt 明确格式要求，解析时容错处理 |
| 对话历史过长超出 token 限制 | 阶段1暂不处理，后续可压缩历史 |
| JSON 文件损坏 | 提示用户备份，初版接受风险 |
| 信息收集判断不准 | Prompt 中明确判断标准，初版接受一定误差 |
| 章节内容不一致 | 审核Agent检查，必要时重写 |
| 生成内容过长被截断 | 分段生成，控制每次输出长度 |
