# v0.6.1 智能体优化设计

## 概述

v0.6.1 是一个优化版本，重点优化智能体架构和 prompt 设计，以提升小说输出质量。

### 核心问题

| 问题 | 表现 |
|------|------|
| 架构冗余 | 存在 CLI 和 Web 两套智能体系统，代码重复 |
| 情节连贯性差 | 前后章节逻辑不通、人物性格不一致 |
| AI 味重 | 文字过于书面化、缺乏细节和情感张力 |
| 大纲质量低 | 情节节点设计不合理、冲突不够吸引人 |
| 审核功能闲置 | 有审核智能体定义但未集成到工作流 |

### 优化目标

1. 移除 CLI 版本代码，统一为 Web 版本
2. 精简智能体数量，从 7 个优化为 5 个核心智能体
3. 优化 prompt 设计，提升小说质量
4. 集成审核重写功能，支持用户选择审核模式

## 设计方案

### 1. 新智能体架构

```
┌─────────────────────────────────────────────────────────────┐
│                      创作工作流                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   灵感表单 ──► 大纲生成 ──► 章节纲生成 ──► 章节正文生成     │
│      │           │             │              │            │
│      │           ▼             ▼              ▼            │
│      │      [大纲智能体]   [章节纲智能体]   [写作智能体]     │
│      │                                         │            │
│      │                                         ▼            │
│      │                              ┌──────────────────┐   │
│      │                              │ 用户选择审核？   │   │
│      │                              └────────┬─────────┘   │
│      │                                       │             │
│      │                              ┌────────▼─────────┐   │
│      │                              │ 审核智能体       │   │
│      │                              │ 不通过？重写(N次)│   │
│      │                              └──────────────────┘   │
│      │                                       │             │
│      ▼                                       ▼             │
│   collected_info                    审核通过 ──► 保存       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2. 智能体职责定义

| 智能体 | 职责 | 输入 | 输出 |
|--------|------|------|------|
| **outline** | 生成结构化大纲 | collected_info（前端表单收集的灵感数据） | 标题、概述、人物设定、世界观、情节节点、情感曲线 |
| **chapter_outline** | 生成章节大纲 | 大纲 + 前文章节纲 | 章节名、场景、人物、情节、冲突、转折、钩子、衔接、结局 |
| **writing** | 写章节正文 | 章节纲 + 前文内容 + 设定 | 章节正文 |
| **review** | 审核质量 | 章节正文 + 章节纲 + 设定 | 通过/不通过 + 分项评分 + 问题列表 + 修改建议 |
| **rewrite** | 重写正文 | 原文 + 审核反馈 | 重写后的正文 |

### 3. 移除的智能体

| 智能体 | 移除原因 |
|--------|----------|
| info_collection | 前端灵感表单已完整收集信息，不需要智能体 |
| chapter_count_suggestion | 纯计算逻辑，硬编码在业务代码中，不需要 prompt |

## Prompt 设计

### 1. outline（大纲智能体）

**输出结构**：

```
标题：《xxx》

概述：[200-300字故事核心]

人物设定：
- 主角：姓名 + 性格 + 核心动机 + 成长弧线
- 配角1：姓名 + 与主角关系 + 作用
- 配角2：...

世界观：
- 时代背景：...
- 核心设定：...
- 力量体系/社会规则：...

情节节点：
1. 开篇：[事件] - [冲突] - [钩子]
2. 发展：...
3. 高潮：...
4. 结局：...

情感曲线：[开篇情绪] → [中段转折] → [高潮] → [结局情绪]
```

**优化点**：
- 要求输出人物关系图，确保人物互动逻辑清晰
- 要求每个情节节点包含「冲突」和「钩子」设计
- 要求设计情感曲线，避免平铺直叙

### 2. chapter_outline（章节纲智能体）

**输出结构**：

```
章节名：[吸引人的标题]

场景：[具体地点 + 环境氛围]

人物：[出场人物 + 本章节的情绪状态]

情节：[本章主要情节，100-200字]

冲突：[本章的冲突/矛盾]

转折：[本章的关键转折点]

钩子：[吸引读者继续阅读的设计]

衔接：[与下一章的衔接点]

结局：[本章收尾/悬念]
```

**优化点**：
- 增加「前文章节概要」输入，强化上下文
- 增加「钩子」字段，设计章节结尾吸引力
- 增加「衔接」字段，规划下一章过渡

### 3. writing（写作智能体）

**写作原则 Prompt**：

```
写作原则：
1. 【展示而非讲述】
   - ❌ "他很生气"
   - ✅ "他握紧拳头，指节发白"

2. 【对话技巧】
   - 对话要有潜台词，不直白说教
   - 每个人物有独特的说话风格

3. 【节奏控制】
   - 紧张场景用短句，舒缓场景用长句
   - 适当留白，不面面俱到

4. 【细节描写】
   - 用五感描写：视觉、听觉、嗅觉、触觉、味觉
   - 细节要服务于情节和人物

5. 【情感张力】
   - 每个场景都要有情感目标
   - 通过冲突和选择展现人物内心

6. 【避免 AI 味】
   - 不用"不禁"、"竟然"、"居然"等书面化词汇
   - 不用"心中涌起一股..."等模板句式
   - 不用"..."过度省略，要写出具体内容
```

**输入增强**：
- 增加「前文章节结尾」（保持连贯）
- 增加「人物关系」和「世界观细节」（保持一致性）

### 4. review（审核智能体）

**输出结构**：

```
【审核结果】通过/不通过

【分项评分】（1-10分）
- 情节一致性：X/10
- 人物一致性：X/10
- 文笔质量：X/10
- 情感张力：X/10
- AI味程度：X/10（越低越好）

【问题列表】
1. [问题描述 + 具体位置]
2. ...

【修改建议】
[具体、可操作的建议]

【通过阈值】各维度 ≥ 6 分，AI味 ≤ 3 分
```

### 5. rewrite（重写智能体）

**重写原则 Prompt**：

```
重写原则：
1. 只修改审核指出的问题部分
2. 保持其他内容的连贯性
3. 不要引入新问题

重写策略：
- 如果是情节不一致：调整细节使其符合设定
- 如果是人物不一致：修正人物言行
- 如果是 AI 味：替换书面化表达，增加细节
- 如果是情感不足：增加心理描写和细节
```

## 审核重写工作流

### 审核模式

| 模式 | 说明 |
|------|------|
| off | 不审核，直接保存 |
| manual | 审核后展示结果，用户决定是否重写 |
| auto | 自动审核，不通过则重写（最多 N 次） |

### auto 模式流程

```
生成章节正文
    │
    ▼
调用审核智能体
    │
    ▼
┌──────────────┐
│ 审核通过？   │──是──► 保存章节
└──────┬───────┘
       │否
       ▼
┌──────────────┐
│ 重写次数 < N？│──否──► 保存 + 标记需审核
└──────┬───────┘
       │是
       ▼
调用重写智能体 ──► 返回审核
```

## 预期效果

1. **代码简化**：移除冗余代码，降低维护成本
2. **小说质量提升**：
   - 大纲结构更完整，人物关系清晰
   - 章节连贯性增强，钩子设计提升吸引力
   - AI 味减少，文笔更自然
3. **审核可用**：用户可选择开启审核重写，提升章节质量
