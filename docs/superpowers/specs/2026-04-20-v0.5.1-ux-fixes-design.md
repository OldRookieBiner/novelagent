# v0.5.1 UX Fixes Design

## Overview

优化小说生成 5 个步骤的使用体验，修复 4 个已确认问题。

## Problems

| # | 问题 | 影响 |
|---|------|------|
| 1 | 审核步骤没有章节列表 | 用户无法快速切换章节审核 |
| 2 | 审核步骤无法点击导航栏切换 | 用户被困在当前页面 |
| 3 | 删除项目使用浏览器原生确认弹窗 | UI 风格不一致 |
| 4 | 导航栏切换到其他步骤无效 | 与问题 2 同根因 |

## Solutions

### 问题1：审核步骤添加章节列表

**修改文件**: `frontend/src/pages/Reading.tsx`

**方案**: 参考 `Writing.tsx:237-265` 实现，在 Reading 页面左侧添加章节列表。

**布局**:
```
+------------------+------------------------+
|章节列表          | 章节内容区|
| - 第1章 (当前)   | 标题、字数|
| - 第2章          | 正文内容|
| - 第3章          | 审核结果|
|                  | 操作按钮|
+------------------+------------------------+
```

**交互**:
- 当前章节高亮
- 点击其他章节跳转到对应审核页面

---

### 问题2+4：导航栏实现跨页面跳转

**修改文件**:
- `frontend/src/pages/Reading.tsx`
- `frontend/src/pages/Writing.tsx`

**方案**: 点击导航节点时，跳转到 `ProjectDetail` 页面并显示对应步骤内容。

**逻辑**:
```typescript
const handleViewStep = (stepIndex: number) => {
  // 跳转到 ProjectDetail，通过 URL 参数指定查看的步骤
  navigate(`/project/${id}?viewStep=${stepIndex}`)
}
```

**ProjectDetail 需要支持**:
- 读取 URL 参数 `viewStep`
- 设置 `viewingStep` 状态显示对应步骤内容

---

### 问题3：删除确认弹窗

**新增组件**: `frontend/src/components/ui/alert-dialog.tsx`

**修改文件**: `frontend/src/pages/Home.tsx`

**方案**: 使用 shadcn/ui 的 AlertDialog 组件替换原生 `confirm()`。

**弹窗内容**:
- 标题: "删除项目"
- 内容: "确定要删除项目「{项目名称}」吗？此操作不可撤销。"
- 按钮: [取消] [确认删除]
