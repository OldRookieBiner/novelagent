# AI 生成预览页面设计

## 背景

当前 AI 生成章节正文时，内容直接显示在富文本编辑器中，格式转换问题多，用户体验不佳。需要重构为预览+编辑分离的模式。

## 目标

- AI 生成时显示预览页面，不直接进入编辑器
- 生成完毕后用户点击"编辑"按钮才进入富文本编辑器
- 支持快速切换章节查看

## 设计方案

### 布局结构

```
┌─────────────────────────────────────────────────────────┐
│ Header                                                   │
├──────────┬──────────────────────────────────────────────┤
│ 章节列表  │ 右上：章节大纲                                │
│ (200px)  │ 场景 | 人物 | 情节 | 冲突 | 结局              │
│          ├──────────────────────────────────────────────┤
│ 第1章 ✓  │ 右下：内容预览                                │
│ 第2章 ✓  │                                              │
│ 第3章    │ [段落内容显示区域]                            │
│ ...      │                                              │
│          │ [状态栏：生成中/已完成 | 字数]                 │
│ 第10章   │ [编辑] [重新生成] [审核]                      │
└──────────┴──────────────────────────────────────────────┘
```

### 左侧面板（200px）

- 显示所有章节列表
- 已生成章节显示 ✓ 标记
- 当前选中章节高亮显示
- 点击章节切换预览内容

### 右上区域：章节大纲

- 显示当前章节的大纲信息
- 字段：场景、人物、情节、冲突、结局
- 紧凑的两列布局

### 右下区域：内容预览

- 纯文本段落显示（无富文本编辑器）
- 段落首行缩进 2em
- 状态栏显示：生成状态 + 字数统计
- 操作按钮：
  - 生成中：停止生成
  - 完成后：编辑 / 重新生成 / 审核

### 交互流程

1. 用户点击"AI 生成" → 进入预览页面
2. 内容实时流式显示在预览区
3. 生成完毕 → 显示"编辑"按钮
4. 点击"编辑" → 跳转到富文本编辑器页面

## 文件修改

| 文件 | 修改内容 |
|------|----------|
| `frontend/src/pages/Writing.tsx` | 重构为预览+编辑双模式 |
| `frontend/src/components/common/TipTapEditor.tsx` | 仅用于编辑模式 |

## Why

用户反馈 AI 生成时富文本编辑器格式显示问题多。分离预览和编辑可以：
- 预览时专注内容，无需处理格式转换
- 编辑时已确认内容，再进入富文本编辑器修改

## How to apply

实现时遵循项目现有的 shadcn/ui + Tailwind 样式规范，保持视觉一致性。左侧章节列表复用 ProjectDetail.tsx 中已有的章节列表组件样式。
